Difference between “ == “ and “ === “ operators.

    Both are comparison operator.
    they return the boolean value
    == is used to compare the value
    === is used to compare the value as well as datatype
    ex. var x = 2 and var y = "2"
    x==y  True
    x===y False

    console.log(null==undefined) //Output:true
    console.log(null===undefined) //Output:false

    console.log("0"==false) //Output:true
    console.log("0"===false) //Output:false

What is execution context

    There are two types of execution contexts: global and function. 
    The global execution context is created when a JavaScript script first starts to run, 
    and it represents the global scope in JavaScript. A function execution context is created whenever a
     function is called, representing the function's local scope.

Explain call(), apply() and, bind() methods. Give an example of call(), apply(), bind()

    Call()
    It is predefined javascripy method.
    It takes argumrnt seperately.
    With call() method one object can use the method or function of another object.
    const obj1={
        name: "raj",
        id:2,
        habt:function(){
            console.log(`${this.name} and ${this.id}`);
        }
    }
    const obj2={
        name: "anu",
        id:5,
        
    }
    obj1.habt.call(obj2)

    Bind()
    It binds a method in a different function
    example:
    const obj1={
        name: "raj",
        id:2,
        habt:function(){
            console.log(`${this.name} and ${this.id}`);
        }
    }
    let newFun = obj1.habt.bind(obj1)
    newFun();

    Apply()
    It is similar to call method
    It takes argumrnt in array.
    const obj1={
        name: "raj",

    }

        name: "anu",
